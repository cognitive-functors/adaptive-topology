# Мета-паттерн FRA: Мульти-доменная таблица изоморфизмов

## Fingerprint → Route → Adapt — один паттерн, бесконечное число воплощений

**Авторы:** Илья Селютин, Николай Ковалев
**Статус:** Рабочий документ
**Дата:** Февраль 2026

---

## TL;DR

Паттерн **Fingerprint → Route → Adapt** (FRA) — это мета-структура, которая проявляется в десятках независимых систем. Но в каждом домене он *называется по-своему*. Эта таблица показывает: за разными словами стоит один и тот же механизм.

## 1. МАСТЕР-ТАБЛИЦА: FRA сквозь домены

### Обозначения столбцов

| Столбец | Значение |
|---------|----------|
| **Домен** | Область знания |
| **Система** | Конкретная система или механизм |

## 2. МЕТА-АНАЛИЗ: ЧТО ВИДНО В ТАБЛИЦЕ

### 2.1 Универсальная трёхфазная структура

Несмотря на радикально разные домены, **все** системы реализуют три фазы:

```
ВХОД → [СЧИТАТЬ СТРУКТУРУ] → [ВЫБРАТЬ СТРАТЕГИЮ] → [ВЫПОЛНИТЬ С КОРРЕКЦИЕЙ] → ВЫХОД
         ↑                      ↑                     ↑
      Fingerprint             Route                  Adapt
```

| Биология | Recognize / Sense / Signal | Select / Move / Transcribe | Mature / Desensitize / Feedback |

| Экология | Survey / Sense / Assess | Specialize / Migrate / Forage | Plasticity / Stopover / Switch |
| Инженерия | Measure / Perceive / Observe | Correct / Decide / Respond | Autotune / Steer / Improve |
| Образование | Diagnose / Assess / Probe | Prescribe / Support / Question | Adjust / Fade / Follow |

### 2.3 Наблюдение: три универсальных глагола

Если свернуть все домены до максимально абстрактных глаголов:

| Фаза | Универсальный глагол | Суть |
|------|----------------------|------|
| **F** | **Считать** (Read) | Извлечь структурную информацию из входа |
| **R** | **Направить** (Direct) | Выбрать действие на основе считанной структуры |
| **A** | **Уточнить** (Refine) | Скорректировать действие по результату |

Или ещё короче: **Понять → Выбрать → Улучшить**.

## 3. ПОЧЕМУ ЭТО ОДИН ПАТТЕРН, А НЕ АНАЛОГИЯ

Можно возразить: «Это просто тривиальная декомпозиция любого процесса на вход → обработку → выход». Но это не так:

1. **Не любая система реализует FRA.** Контрпример: *фиксированная стратегия* (делать одно и то же независимо от входа) — нет fingerprint, нет routing. Жёсткий алгоритм — не FRA.

2. **Теорема доказывает превосходство.** FRA *математически гарантированно* ≥ фиксированной стратегии (Theorem 1, Partitioning Bound). Это не мнение — это неравенство.

3. **Гетерогенность — ключевое условие.** FRA даёт *строгое* преимущество тогда и только тогда, когда разные стратегии оптимальны для разных подклассов входов (Theorem 2, Heterogeneity Bound). Это тестируемое условие.

4. **Конвергентная эволюция.** 32+ систем из 6 доменов *независимо* пришли к одной структуре. Это не заимствование — это математическая неизбежность в гетерогенных средах.
